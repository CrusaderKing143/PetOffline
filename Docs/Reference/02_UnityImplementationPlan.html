01

不可违反的架构契约

这一节直接解决“HTML Canvas 能跑，但 Unity 不能把一切都做成 UI”的问题。

正确契约：World owns gameplay; UGUI presents state.

Game 主场景拥有

Transform、SpriteRenderer、Rigidbody2D、Collider2D、摄像头感知、触发区、机器人、任务物品、角色动画、世界音效。

UGUI 拥有

HUD、任务文字、进度条、视频窗、字幕、通知、报告、最终选择、暂停、设置、屏幕淡入淡出。

桥接方式

世界广播只读事件 / ViewModel；UI 发送高层 Command。两边禁止直接持有对方内部组件。

UGUI / Screen Space

RenderTexture 的正确位置

只有“真实摄像头画面”或房间内电视需要 RenderTexture。它是一块世界/UGUI表面上的内容来源，不是用来把整个游戏世界塞回 Canvas。

推荐

  • 主人视频窗：UGUI 头像或预录序列帧。
  • 会议电视：世界 SpriteRenderer / Quad + RenderTexture。
  • 摄像头 A 抓拍：独立 CaptureCamera，仅在报告需要截图时启用。

不推荐

  • MainCamera 渲染到 RawImage，再在 RawImage 上做全部交互。
  • 用 GraphicRaycaster 替代 Physics2D 世界交互。

// Core：UI 与 Gameplay 共享的只读状态
public interface ILevelViewModel
{
    string Objective { get; }
    float Objective01 { get; }
    CameraUiState CameraState { get; }
    event Action Changed;
}

public interface ILevelCommandSink
{
    void SubmitBark();
    void SubmitChoice(FinalChoice choice);
}
// UI：只显示，不碰世界 Transform
public sealed class LevelHudPresenter : MonoBehaviour
{
    ILevelViewModel model;

    public void Bind(ILevelViewModel value)
    {
        Unbind(); model = value;
        model.Changed += Refresh;
        Refresh();
    }

    void Refresh()
    {
        objectiveLabel.text = model.Objective;
        progress.value = model.Objective01;
    }
}

09

从 HTML Canvas 原型迁移到 Unity

迁移的是规则、参数和体验，不是绘制方式。

概念图本身已天然区分世界画面和 HUD。Unity 应沿用这个视觉关系，而不是把公寓也做成 UI。

映射表

HTML原型Unity正确实现
Canvas draw floor/furnitureTilemap / SpriteRenderer GameObjects
player.x / player.yTransform.position + Rigidbody2D
drawCones()VisionConeRenderer + CameraVisionSensor2D
circle/rect hit testCollider2D / Trigger / Physics2D Query
DOM task listUGUI TaskPanel + Presenter
JS phase string强类型 Enum 状态机
setTimeout dialogueDialogueDirector / Timeline / Coroutine
Canvas depth sortSortingGroup + Y Sort
迁移红线：不要创建一个 1920×1080 Canvas，再把每件家具和角色做成 Image；不要把鼠标点击地面做成 GraphicRaycaster；不要让 UI 动画代替世界物理。这样虽然能“看起来像原型”，但无法形成可维护的 Unity 游戏。

12

性能、构建与交接

最终包应当在无网络、无编辑器环境下完整运行。

性能预算

  • 世界相机 1 个常驻；CaptureCamera 仅需要时启用。
  • 摄像头感知 10Hz 或事件采样。
  • 粒子与世界 UI 使用对象池。
  • 无运行时生成大纹理。
  • 避免 Update 中 LINQ 与字符串拼接。

构建门槛

  • Windows x64 Development / Release 各一份。
  • Player.log 无异常。
  • 从标题到两个结局均可到达。
  • 键位、音量、返回标题可用。
  • 丢失焦点与暂停恢复稳定。

交付内容

  • Unity 工程与锁定版本。
  • 可执行 Build。
  • README / Controls / Known Issues。
  • 任务与状态机图。
  • UI/Game 分层审计结果。
  • 测试清单与回归记录。
Definition of Done：两关流程完整、世界玩法不依赖 UGUI、UI 不拥有世界状态、所有核心任务均有自动或手动回归测试、推荐结局的情绪演出可稳定复现。
